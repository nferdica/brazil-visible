# Brazil Visible — Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Build a Docusaurus documentation site cataloging 93 Brazilian public data sources with cross-referencing recipes for government oversight.

**Architecture:** Docusaurus static site with structured YAML frontmatter in Markdown files. Each of the 93 data sources gets a dedicated page organized by category (21 folders). A separate section documents cross-referencing recipes. PT-BR primary language with i18n support for English.

**Tech Stack:** Docusaurus 3.x, React, TypeScript, MDX, GitHub Actions, GitHub Pages

# Brazil Visible — Claude Code Instructions

## Project Overview
Documentation site (Docusaurus) cataloging 93+ Brazilian public data sources.

## Structure
- `docs/apis/<category>/<source>.md` — API documentation pages
- `docs/cruzamentos/<recipe>.md` — cross-referencing recipes
- `src/pages/` — custom pages (landing page)
- `i18n/en/` — English translations

- All API pages MUST have the required frontmatter fields (title, slug, orgao, url_base, tipo_acesso, status)
- Content in PT-BR first, English translations in i18n/en/
- Commit messages follow conventional commits (feat:, docs:, ci:, fix:)
- Branch from `develop`, PR to `develop`, merge `develop` to `main` for releases

## Adding a New API
1. Create `docs/apis/<category>/<slug>.md`
2. Fill frontmatter with all required fields

1. Create `docs/cruzamentos/<slug>.md`
2. Fill frontmatter (title, dificuldade, fontes_utilizadas, campos_ponte, tags)
